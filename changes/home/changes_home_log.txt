Cambios realizados para integrar el Home web (2025-10-24):

Resumen general:
- Integré tu código web (Home React/JSX) como una página HTML local dentro de la app Android, y la cargo usando un WebView en `HomeActivity`.
- Objetivo: que al iniciar sesión el usuario vea la UI exactamente como en tu página web (hero + catálogo + footer), sin migrar a React Native o rehacer el UI en nativo.

Archivos añadidos / modificados:

   - `index.html`  -- Contiene tu Home en JSX + Tailwind vía CDN; transpila JSX en runtime con Babel (prototipo).
     - Comentarios: dentro de `index.html` verás la marca `//jere` en varios lugares para indicar integraciones específicas con Android.
     - Nota: para producción conviene compilar el JS fuera y eliminar Babel en runtime.

2) Archivos nativos añadidos/modificados:

Qué implementé y por qué (detalles):
- Enfoque: WebView local (archivo en `assets/www/index.html`) para mantener exactamente el look & feel de tu web sin reescribir todo en Android.
- Comunicación web -> Android: `webView.addJavascriptInterface(AndroidBridge(), "Android")` expone `Android.onReserve(serviceJson)` para que al pulsar "Comprar Ahora" en la web la app reciba el JSON (actualmente registrado en logs).
- Comunicación Android -> web: paso el nombre del usuario como query param `?user=...` al cargar el index.html; la web extrae `user` de `window.location.search` y lo usa para mostrar estado autenticado. (Ver `index.html` en assets).
- Seguridad y notas:
  - Para producción no uses `Babel` en runtime ni Tailwind CDN; bundle tu app (build) y sirve los archivos minificados.
  - `addJavascriptInterface` puede ser peligroso si la WebView carga contenidos externos; como estamos cargando assets locales es aceptable.

Cómo probar localmente (método rápido):
1) Compila y ejecuta la app desde Android Studio.
2) Registra un usuario en `RegisterActivity` y luego haz login.
3) Al hacer login con éxito, `LoginActivity` abre `HomeActivity` y ahora cargará tu UI web desde `assets/www/index.html` en el WebView. El nombre del usuario se envía en la URL y la web lo mostrará (si existe).
4) Desde la web, pulsar "Comprar Ahora" dispara `window.Android.onReserve(JSON.stringify(service))` — revisa Logcat para ver el mensaje: `Reserva recibida desde Web: ...`.

Comentarios en el código (marcas //jere):
- Dentro de `index.html` y `HomeActivity.kt` usé comentarios `//jere` para indicar puntos de integración realizados por el equipo.
